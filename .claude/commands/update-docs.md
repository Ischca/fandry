# ドキュメント一括更新

コードに変更があった場合、関連するすべてのドキュメントを更新してください。

## 更新対象ファイル

1. **todo.md** - タスク管理
2. **CLAUDE.md** - 開発者向けガイド
3. **README.md** - プロジェクト説明

## 更新手順

### Step 1: 変更内容の確認
```bash
git status
git diff --stat
git log --oneline -5
```

### Step 2: todo.md
- 完了したタスクを[x]でマーク
- 新しいタスクがあれば追加
- 不要になったタスクは削除

### Step 3: CLAUDE.md
アーキテクチャに変更があった場合のみ:
- 新しいルーター/API
- スキーマ変更
- 重要なコンポーネント追加
- コマンドや環境変数の変更

### Step 4: README.md
ユーザー向け機能に変更があった場合のみ:
- 新機能の説明
- 技術スタックの更新
- セットアップ手順の変更

## 注意事項

- 変更に関連するドキュメントのみ更新する
- 過度な更新は避ける（軽微な変更は無視してよい）
- 各ドキュメントの対象読者を意識する:
  - todo.md → 開発チーム
  - CLAUDE.md → 開発者/AI
  - README.md → すべてのユーザー
