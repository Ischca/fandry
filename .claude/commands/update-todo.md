# TODOリストの更新

現在のコードベースの状態を確認し、todo.mdを適切に更新してください。

## 手順

1. **現在のtodo.mdを読む**: `todo.md`の内容を確認
2. **最近の変更を確認**: git diffやgit logで最近の変更を把握
3. **完了項目を更新**: 実装済みの機能を「完了済み」セクションに移動
4. **新規タスクを追加**: 会話中に発見した新しいタスクを追加
5. **優先度を調整**: 残りのタスクの優先度を見直し

## フォーマット

```markdown
## 完了済み機能
- [x] 機能名 (完了日または詳細)

## 進行中・今後の実装
- [ ] 機能名

## 技術的負債・改善項目
- [ ] 改善項目
```

## 注意事項

- 具体的で実行可能なタスク記述にする
- 関連するタスクはグループ化する
- 完了したタスクは必ず[x]でマークする
- 進行中のタスクがあれば「進行中」と明記する
